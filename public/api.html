ROBIN NFT Radar · API 傻瓜说明书

用这份文档，你可以在 自己的脚本、电报机器人、本地 AI 代理里读取 Robinhood Chain 上的铸造热榜数据——不必一直开着网页。网站给人看；API 给机器用。

① 通过本 API 能获取哪些服务？

服务名建议对外统称：ROBIN NFT Radar（或 RobinRadar）。

服务能力 说明 主要接口
铸造热榜 近 1 小时 mint 最多的集合；含 5m / 30m / 1h 数量、铸造单价等 GET /api/mints → hot
山巅之王 当前 1 小时 mint 数量第 1 名 GET /api/mints → king
实时铸造流 最近发生的 ERC-721 mint 明细 GET /api/mints → feed
已铸完 + 二级 售罄合集；尽量提供 OpenSea 地板价、成交额 GET /api/mints → mintedOut
服务健康 是否在线、轮询是否正常 GET /api/health、/api/status
钱包 NFT 查询某地址在 Robinhood 链上的 NFT 持仓 GET /api/wallet/nfts
不会提供：代下单、私钥托管、保证收益。数据有延迟，价格以最新一笔 mint 单价为准，可能短暂显示为空（查询中）。

③ 接口一览（给开发 / 机器人）

GET /api/health

探活。返回纯文本 ok。

curl -s https://robinradar.xyz/api/health

GET /api/status

服务状态：轮询是否健康、内存、是否配置 OpenSea 等。适合监控「数据源挂了没」。

curl -s https://robinradar.xyz/api/status

GET /api/mints ★ 主接口

一次拿齐热榜、流、已铸完、山巅之王。

参数 默认 说明
window 60 热榜时间窗（分钟，约 1–60）
hot 30 热榜返回条数
feed 100 实时流条数
out 50 已铸完条数；0 = 不要已铸完数组 
curl -s "https://robinradar.xyz/api/mints?hot=10&feed=5&out=5&window=60"

响应里最常用字段：

JSON 路径 含义
king 山巅之王（1h mint 第 1）
hot[].name 合集名
hot[].contract 合约地址
hot[].mints1h / mints5m 1 小时 / 5 分钟 mint 次数
hot[].priceDisplay 最新 mint 单价文案（如 Free / 0.001 ETH）
hot[].explorerToken 区块浏览器链接
hot[].opensea OpenSea 链接（若有）
mintedOut[].tradeVolumeDisplay 已铸完成交额
mintedOut[].floorPriceDisplay 已铸完地板价

GET /api/wallet/nfts?address=0x...

查询某地址在 Robinhood 链上的 NFT 列表。

curl -s "https://robinradar.xyz/api/wallet/nfts?address=0x你的地址"

GET /api/raffles

社区白名单抽奖期数（公开只读）。

  • MCP / HTTP 工具：若支持「原始 HTTP」，注册 GET /api/mints 即可。
步骤 3：用自然语言让 AI 调用

见下一节「一句话示例」。成功时 AI 应先请求接口，再总结名称、地址、链接。

可复制配置块（贴进 AI 规则 / Skill）

# 工具：ROBIN NFT Radar（RobinRadar）

## 何时使用
用户问到：Robinhood 链 NFT 铸造、热榜、谁在 mint、山巅之王、已铸完、地板价/成交额、
合约地址、OpenSea/浏览器链接时，使用本工具。

## 如何调用
Base URL: https://robinradar.xyz
主接口: GET /api/mints?hot=15&feed=10&out=10&window=60
探活: GET /api/health
状态: GET /api/status

请用 HTTP GET 获取 JSON，不要编造链上数据。
若 priceDisplay 为空，说明单价仍在查询，如实告知用户。

## 回答时尽量包含
- 合集名称 name
- 合约 contract
- 1h/5m mint 数量
- 价格 priceDisplay（若有）
- 链接 explorerToken 或 opensea
- 山巅之王 king（若问热度第一）

## 示例用户话
「帮我看看一小时内正在大量铸造的 NFT」
「现在山巅之王是谁，给我合约和 OpenSea」
「已铸完里成交额高的有哪些」

JavaScript 一行拉取（脚本 / 自定义工具）

const r = await fetch("https://robinradar.xyz/api/mints?hot=10&feed=0&out=0");
const data = await r.json();
console.log(data.king, data.hot);

⑥ 注意与限流（请礼貌使用）

  • 建议请求间隔 ≥ 5～10 秒；同一数据可本地缓存，无需每秒打。
  • 数据来自链上索引，有数秒～数十秒延迟；铸造价可能先显示为空再补齐。
  • 非投资建议；合约有风险，请自行验证。
  • 滥用高频可能导致临时限流；日后可能增加 API Key 与付费档。
  • 网站 UI 继续免费开放：雷达面板。

合作 / 更高配额 / 电报告警：见网站 Twitter 入口。
